What is Command and Control (C2) ?

A C2 works like this, a devices will requesting command (beaconing) to C2 server where we put command as bot operator then the C2 will give response a command that your devices need to execute.

C2 is a infrastucture (program + server) used to communicate with your devices remotely for maintaining access. C2 itself work in client-server architecture, mean there a C2 server and C2 client (implant) - C2 client in this context is your devices.

The C2 server is a server the used to supply command to devices and C2 client is a program that beaconing to C2 server to request and execute command that supplied. C2 client supposed to running in device you want to access.

Providence is C2 server only, which mean you need implement C2 client by your own. Maybe in future I will provide example of client.

What the general-purposeness in Providence ?

C2 widely known for it malicious intent, thats why ATT&CK MITRE become good references for detecting C2 presence. C2 could be use for command base for botnet army, control center in cyberwarfare, and much more.

This not mean C2 does not have good purpose in it existential mean. There are example to C2 used for good purpose, like parental app control, biling system at internet cafe, remote access tool and many other.

Base example above you could choose what intention C2 client you want implement, Providence works as message broker and report panel. The general purposed of Providence is adaptable of C2 to be match with client design that provide way to communicate with operator, client-centric C2 server.

In simple way, you just design the program you want then connect it with Providence. Providence recently only support gRPC, because it schema design so you just generate client code and use it in your program.

Futher, I will add HTTP/S support and adding some new communitcation channel for open opportunity to advance feature like file transaction or data streaming.

Why there 3 modes to run Providence ?

The main reason is scalability, you could have many proxy mode instance for load balancing and run single panel mode instance to conect to one of the proxy and set command.

The full mode intented if you want run full-spectrum of Providence in one-go, like in local network or else.
